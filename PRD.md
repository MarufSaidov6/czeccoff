# PRD: Flash Reader — RSVP-читалка

**Версия:** 1.0 (зафиксировано состояние продукта) · **Дата:** 2026-06-10 · **Автор:** Маруф + Claude
**Артефакт:** `speed-reader/index.html` — одностраничное web-приложение без зависимостей и сборки, открывается двойным кликом.

---

## 1. Суть продукта

Читалка для быстрого чтения методом RSVP (Rapid Serial Visual Presentation): слова показываются по одному в фиксированной точке экрана, якорная буква (ORP — Optimal Recognition Point) подсвечена красным и всегда стоит в одной позиции. Глаза не совершают саккад — скорость чтения растёт с ~250 до 400–700 слов/мин.

Ключевое отличие от аналогов (Spritz, Reedy): продукт борется с главной слабостью RSVP — потерей понимания на скорости. Для этого: умный темп, перемотка по предложениям, карта регрессий, адаптивный тренер.

**Гипотеза:** RSVP + умный темп + инструменты возврата = чтение в 2 раза быстрее без субъективной потери понимания и с возвратом на вторую сессию.

## 2. Тип проекта и пользователь

Портфолио-проект / эксперимент, ведётся по design thinking с реальными циклами тест → находка → правка (см. раздел 6 — журнал решений).

**Персона:** «занятый читающий», 25–40, нон-фикшн и рабочая литература на русском и английском, список «прочитать» растёт быстрее, чем читается. Основной контент — книги в fb2 и txt (включая «грязные» файлы с lib.ru).

**Анти-персона:** читатель художественной прозы ради удовольствия ритма — RSVP не для этого кейса.

## 3. Что реализовано (текущая правда)

### Ридер
- Одно слово на экран, якорная буква красным, метки-направляющие сверху/снизу.
- **Пиксельный ORP-алгоритм** (как у Spritz, а не упрощённый индексный): слово измеряется canvas-метрикой в текущем шрифте, якорь — буква, чей центр глифа ближе всего к 30% пиксельной ширины слова. Учитывает реальную ширину букв («ш» ≠ «т») и выбранный шрифт.
- **Три профиля позиции якоря** (настройка): 24% / 30% (классика) / 36% — инструмент калибровки ORP под кириллицу; у английской таблицы Spritz нет данных для русского, подбираем эмпирически.
- Раскладка `minmax(0,1fr) auto minmax(0,1fr)`: якорь геометрически в центре всегда, длинные слова переполняются наружу, автоужатие шрифта считается от большего «полукрыла» слова относительно якоря.
- NFC-нормализация входа (разложенные «й»/«ё» не ломают якорь), поддержка суррогатных пар.

### Умный темп
Паузы относительно базового темпа: конец предложения +140%, запятая/двоеточие +60%, граница абзаца +130% (удержание последнего слова, отключаемо), числа +100%, длинные слова +30..60%, начало реплики +80%. Плавный разгон после паузы (4 слова). Отключается в настройках.

### Смена слов — три научных режима
- **Резкая** (дефолт) — стандарт RSVP;
- **Микро-промежуток** — пустой зазор ~15% длительности (до 60 мс) перед следующим словом: снижает взаимную зрительную маскировку слов (ISI из RSVP-исследований), рекомендуется на скоростях 500+;
- **Плавная** — лёгкий fade; науке противоречит (увеличивает маскировку), оставлен для честного сравнения.

### Диалоги
Тире склеивается с первым словом реплики («— Привет»), перед репликой пауза-«вдох». Три режима отображения (настройка): тире со словом / индикатор «— реплика» под словом / оба. Решение по дефолту — после сравнения на живом тексте.

### Навигация и контекст
- Перемотка по предложениям (← →), клик по прогресс-бару (прыжок к началу предложения), кнопка «в начало».
- При паузе: текущее предложение целиком с подсветкой позиции + предыдущее бледным (ориентир после перемотки).
- Пауза/перемотка никогда не оставляют на служебном кадре.

### Контент и парсинг (бой с реальными файлами)
- Вставка текста, txt, **fb2** (DOMParser, автодетект кодировки из XML-декларации — старые fb2 в windows-1251 работают; сноски `body name="notes"` отбрасываются).
- Нормализатор распознаёт **три формата абзацев**: пустые строки / отступы lib.ru (абзац = строка с отступом, внутри жёсткие переносы) / «каждая строка — абзац». Проверено на реальном тексте Моэма с lib.ru.
- `--` конвертируется в тире (включая «конверт.-- О!» без пробелов), мягкие дефисы удаляются.
- Токенизатор: сокращения («т.е.», «т.д.», инициалы «А. С.») не рвут предложения; числа с разрядами склеиваются в один кадр («1 587 423»); сверхдлинные дефисные склейки дробятся.

### Тренер скорости и карта регрессий (киллер-фича, ставка продукта)
- Тренер (опционально): +10 сл/мин за чистую минуту чтения без возвратов, −10 за перемотку назад, с тостами. Диапазон 100–900.
- Карта регрессий: каждое место возврата запоминается по книге. На экране финиша — «ты возвращался к N местам» и список этих предложений обычным текстом, сортировка по числу возвратов. Лечит главную боль RSVP — невозможность регрессий.

### Статистика и удержание
- В ридере: заданная скорость и **фактическая** (ниже из-за умных пауз — разница видна и осмысленна), прочитано слов, время, **осталось ~N мин** (по фактической скорости).
- Честный подсчёт: слово засчитывается один раз (фронтир maxIdx), перечитывание не накручивает.
- Статистика по дням: «сегодня» + бары за неделю на главном экране.

### Эргономика
Тёмная тема (фон #16161a, текст #e8e6e1, якорь #ff5a4e — смягчённый красный, не «вибрирует»), фокус-режим (интерфейс тает при чтении), отсчёт-«вдох» перед стартом, размер слова 60–140%, шрифт с засечками/без, клавиатура (пробел, стрелки, Home) и тач, экран финиша со статистикой.

### Хранение
Позиции и настройки — localStorage; текст последней книги — IndexedDB (нет лимита 2 МБ, нет синхронных записей на главном потоке). Сохранение при закрытии вкладки и сворачивании. Кнопка «назад» на мобильном — через history API, не убивает сессию.

## 4. Чего осознанно нет

Аккаунтов и синхронизации, мобильных приложений (web отзывчивый), epub и PDF, встроенного каталога книг, социальных функций. Юридически: пользователь загружает свои файлы, мы ничего не распространяем.

## 5. Архитектурные принципы

- Один HTML-файл, ноль зависимостей — скорость итераций важнее структуры, пока гипотеза не подтверждена.
- Кадры бывают контентные (слова) и служебные — навигация и статистика работают только по контентным.
- Все «ручки» восприятия (позиция якоря, режим смены слов, множители темпа) — настройки, а не константы: продукт является стендом для калибровки самого себя.
- Тестовый стенд: токенизатор и нормализатор прогоняются node-скриптами на коварных корпусах (диалоги lib.ru, сокращения, числа, NFD).

## 6. Журнал ключевых решений (design thinking в действии)

| Итерация | Находка теста | Решение |
|---|---|---|
| 1 | Якорь попадал на точку в «т.е.» | Соскальзывание якоря на ближайшую букву |
| 2 | «Диалоги сложно уловить» (Маруф) | Тире клеится к реплике + пауза + 3 режима отображения |
| 2 | «Не перемотать с начала» (Маруф) | Кликабельный прогресс-бар + кнопка в начало |
| 2→3 | Пауза оставляла на служебном кадре ¶ | Запрет навигации на служебные кадры |
| 3 | Значок ¶ непонятен (Маруф) | Заменён пустым кадром |
| 3 | Пустой кадр читается как зависание (Маруф) | Кадр убран; последнее слово абзаца держится дольше. Урок: отсутствие сигнала = сигнал о сбое |
| 3 | Текст Моэма с lib.ru: абзацы по отступам, тире как `--` | Нормализатор трёх форматов + конвертация тире |
| 3 | Якорь «довольствоваться» уехал из центра (Маруф, скриншот) | minmax-сетка + пиксельный ORP с canvas-метрикой вместо индексного |
| 4 | Автотест TOK-04: «и т.д. Конец.» не закрывал предложение | Сокращение + заглавная следом = конец предложения; инициалы — исключение. Тестовый раннер сохранён в `tests/run.js` (32 кейса) |

## 7. Метрики эксперимента

- **Активация:** ≥ 2 минут непрерывного чтения в первой сессии.
- **Главная:** возврат на вторую сессию (цель ≥ 40%).
- **Понимание:** пересказ после сессии на ×1.5–2 от базовой скорости (интервью, 5–7 тестеров).
- **Комфорт:** медианная сессия ≥ 10 минут; инструмент измерения — дневная статистика и фактическая скорость.

## 8. Бэклог (приоритезированный)

1. **Главы fb2** — оглавление, переход по главам; документ перестаёт быть плоским текстом (отдельная итерация, меняет структуру данных).
2. **Калибровка по данным:** связать профиль якоря и режим смены слов с фактической скоростью/регрессиями — пусть продукт сам найдёт лучшие настройки.
3. PWA (манифест, офлайн, share-target «читать что угодно из браузера»).
4. Библиотека книг (несколько книг в IndexedDB вместо «последней»).
5. Темп по частотности слов (офлайн-словарь: редкие слова медленнее).
6. Квиз на понимание после главы (фаза 3 из исходного PRD).
7. Жесты на мобильном; sepia/светлая тема; вертикальная стабильность якоря при автоужатии.

## 9. Открытые вопросы

1. Вердикт Маруфа по режимам диалогов (тире/индикатор) — выбрать дефолт.
2. Профиль якоря для кириллицы: подтвердится ли сдвиг к центру (36%) на длинных словах?
3. Микро-промежуток: делать ли его автодефолтом на скоростях 500+?
4. Шаг тренера ±10 — не слишком ли агрессивен? Калибровать по ощущениям.
